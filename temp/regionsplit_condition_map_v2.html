「汚染源」の再定義（WL7の知見）：OHWが高い必要はない。GC単位あたりの価値密度 = (1−OHW) / mean_size が低い大サイズコンテンツであれば干渉が発生する。
ケース別図解
ケース1 Block Cache型（強OHW勾配） BMR大幅改善
大サイズほどOHW急上昇。保護対象（中サイズ・中OHW・高バイト）と汚染源（大サイズ・高OHW）が明確に分離できる。
代表: DS3実トレース / WL-01b / WL-06
B×(1-r) = 9〜11 D_size = 4〜5倍 global_α = 0.47〜0.56 local_a = 低〜中 ΔBMR = −0.70〜−0.99%pt
鍵: OHW急上昇境界が閾値候補。保護対象byte% ≥ 10% かつ OHW中程度（50〜80%）が必要。
ケース2 サイズ差のみ型（OHW均一・WL7） BMR改善（新発見）
OHWが全帯域で均一低（20〜26%）でも、GC単位あたり価値密度の差（大サイズ＝密度低）が干渉を生み出す。
代表: WL7（256KB帯byte=43%）
B×(1-r) = 32.8（最大） D_size = 4倍 OHW均一（20〜26%） 大プール負荷率=1.28倍 ΔBMR = −1.08〜−1.23%pt
鍵: OHW高不要。保護対象のbyte%が大きく（43%）、大プールの過負荷率が低い（≤1.3倍）ときに成立。
ケース3 大プール過負荷型（WL8） 中立（過負荷で相殺）
ケース2と同じOHW均一型だが、大サイズ帯のbyte%が増えて大プールが過負荷（負荷率≥1.6倍）。C_partitionが低下し改善が相殺される。
代表: WL8（1MB-2MB byte=30%）
B×(1-r) = 25.2（大） 大プール負荷率 = 1.61倍 C_partition低下 global_α = 0.45 ΔBMR ≈ +0.01〜+0.11%pt
鍵: 大プール負荷率 = 大プールbyte% / pool_ratio。1.5倍超で競合激化。C_partition = max(0.1, 1 − 0.5×(負荷率−1))
ケース4 均一低OHW型（Tencent Block） BMR悪化（大プール内競合）
全帯域OHW均一低（15〜25%）かつ各帯域がほぼ同規模のバイト量。大プール内で異なるサイズが競合し、大サイズが中サイズを排出。
代表: Tencent Block / WL-03
OHW変動幅 < 15%pt C_partition 負 global_α = 0.47〜1.2 ΔBMR = +0.1〜+0.43%pt
鍵: 大プール内で均一低OHWの異サイズが競合。大サイズが中サイズを排出しBMR悪化。Zipf alphaが高いほど悪化が激しい。
ケース5 高local_a型（LRU最適・WL8-like） 中立（LRU済み最適化）
ビン内局所性（local_a）が高く、LRUが各サイズ帯内でホットオブジェクトを自然保持。RegionSplitが追加できる価値が薄い。global_αが低くても高性能。
代表: WL8（a↑, global_α=0.45）
local_a 高（ビン内集中） global_α 低（0.45） LRU MR = 24%（高性能） RegionSplit追加価値なし ΔBMR ≈ 0〜+0.11%pt
鍵: global_α ≠ LRU最適性の指標。ビン内 local_a が高いとLRUが局所最適化 → RegionSplitの追加価値が消える。
ケース6 CDN型（逆転OHW） 中立〜有害
大サイズほどOHW低い（価値ある）。LRUが自然に大サイズを保持。分割しても「汚染源」と「保護対象」が入れ替わり効果なし。
代表: Meta CDN / WL-02
保護すべき「廃棄コンテンツ」なし global_α = 0.7〜0.8 ΔBMR ≈ 0（全閾値）
鍵: OHW勾配が逆向き。大サイズコンテンツはキャッシュに留まるべき価値あるコンテンツ。隔離すると逆効果。
ケース7 二峰性汚染型（MR改善・BMR中立） MR↓ / BMR 中立
小サイズ低OHW（リクエスト多・byte≈0）と大サイズ超高OHW（byte大）の二峰分布。MRとBMRの改善条件が独立していることを示す典型例。
代表: WL-05
ΔMR = −3.0%pt（大） B_protect×(1-r) ≈ 0（byte≈0） ΔBMR ≈ −0.002%pt（≈0）
鍵: MRはリクエスト量依存、BMRはbyte量依存。保護対象のbyte%≈0のためBMRは改善しない。MR改善はbyte改善を保証しない。

ケース2の新判定基準（WL7から導出）： OHW勾配がなくても B×(1−r) ≥ 20 かつ D_size ≥ 4 かつ 大プール負荷率 ≤ 1.3 が揃えばBMR改善が起きる。 「汚染源」はOHW高でなく「GC単位あたり価値密度 = (1−OHW)/mean_size が低い」コンテンツで十分。
global_αとlocal_aの使い分け： global_αはワークロード全体の集中度（LRU有効性の粗い指標）。local_aはビン内集中度（LRU局所最適性の精緻な指標）。local_a高 → ビン内LRU最適 → RegionSplit追加価値小。
