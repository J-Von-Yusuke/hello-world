RegionSplit 閾値判定 — 4 経路 Path 模式図

各 Path がどのようなリクエスト量分布・バイト量分布・OHW 率分布に対して閾値 T を決定するかを示す

リクエスト量分布 (req%)
バイト量分布 (byte%)
OHW率 r
閾値 T (設定位置)
Path A
OHWジャンプ検出
代表: mrsBlk, metaKV, cldphyBlk
検出条件
有意ビンを小→大の順に走査し、r < 0.70 の「低 OHW ゾーン」確認後、 r ≥ 0.75 に急上昇するビンを検出
→ T = 急上昇ビンの下端

分布の特徴
小サイズ: リクエスト集中 + 低 OHW (r < 0.70) ← 宝庫ゾーン
大サイズ: リクエスト分散 + 高 OHW (r ≥ 0.75) ← 汚染ゾーン

なぜ有効か
大サイズ高 OHW オブジェクトが GC-Region を占拠し、小サイズ低 OHW を排出する干渉を遮断する。
Path B
バルク隔離
代表: metaCDN (bin28/29 含む)
検出条件
全ビン中の最大バイト占有ビンの byte_share が > 80%
→ T = そのビンの下端

分布の特徴
単一の巨大サイズビンがバイトの 80% 超を独占。リクエストは少ないが、 容量配分でほぼ全プールを支配する

注意
スコープ外サイズ (例: 256MiB 以上) がバルクを占有している場合、 --max-bin N で除外してから再判定が必要。 除外後はPath D や Path C が適用される。
Path D
OHW V字谷検出
代表: metaCDN (bin28/29 除外後, bin19)
検出条件
有意ビン i で r[i-1] - r[i] ≥ 0.05 かつ r[i+1] - r[i] ≥ 0.05 (V字形)
かつ N_b[i] ≥ 前後平均 N_b × 1.5 (リクエスト異常集中)
→ T = ビン i の下端 (V字谷ビンを大プールへ保護)

分布の特徴
OHW 率の谷 + そのビンへのリクエスト集中が同時に出現する。
谷ビンは非 OHW 再利用も高く、大量の OHW バイトを抱えている

なぜ有効か
V字谷ビンを小プールへ入れると OHW バイトが大量流入し BMR が悪化する。 T をそのビン下端に設定し大プールへ残すことで BMR 悪化を防ぐ。
Path C
CDF 交差点
代表: tncntBlk, wikiCDN
検出条件
小→大の順に累積し、F_req + F_byte ≥ 1.0 を最初に超えるビン
→ T = そのビンの下端

分布の特徴
OHW 率は全サイズで均一低値 (OHWジャンプなし)。
リクエストは小サイズ集中、バイトは大サイズ集中 → 重心が乖離

なぜ有効か
リクエスト重心 (≈ 小サイズ) とバイト重心 (≈ 大サイズ) の交差点が 「高頻度コンテンツと低頻度大コンテンツ」の自然な境界となる。 OHW 汚染ではなく GC-Region 競合による干渉を解消する。
